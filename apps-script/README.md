# Apps Script — Phase 1: 응답 시트 정리/마이그레이션

기존 구글폼 응답 시트를 **그대로 보존**한 채, 오른쪽에 "앱 컬럼"을 자동으로 채워주는 스크립트입니다.
(가족/그룹 복원 · 금액 계산 · "확인필요" 자동 플래그)

## 배포 방법 (사장님/대표 계정에서 1회)

1. 응답 시트 열기 → 상단 메뉴 **[확장 프로그램] → [Apps Script]**
2. 편집기에 `Migrate.gs` 내용을 통째로 붙여넣고 💾 저장
3. 함수 목록에서 **`enrichSheet`** 선택 → **▶ 실행**
4. 최초 1회 **권한 승인** (이 시트를 편집하도록 허용)
5. 시트 오른쪽 끝에 아래 컬럼이 채워집니다. (다시 실행하면 재계산 = 멱등)

> 폼에 새 응답이 계속 들어오므로, 갱신하려면 `enrichSheet`를 다시 실행하거나,
> `installOnFormSubmit`을 1회 실행해 **폼 제출 시 자동 갱신** 트리거를 걸면 됩니다.

## 추가되는 컬럼

| 컬럼 | 의미 |
|---|---|
| 제출경로 | 기존폼 / (앞으로) 앱 |
| 그룹ID | 이메일 기준 가족·그룹 묶음 ID (G001…) |
| 그룹대표(추정) | 대표자칸 → 입금자명 → 장년부 순으로 추정 |
| 그룹인원(제출) | 같은 그룹으로 실제 제출된 인원 |
| 1인등록비 | 그 사람 부서 등록비 |
| 본인버스 | 38,000 또는 0 |
| 본인설악산 | 10,000 또는 0 |
| 그룹공동비용(객실+투숙) | 객실·투숙 추가비용 (그룹당 1회, **대표행에만**) |
| 그룹총액 | 그룹 전체 합계 (**대표행에만**) |
| 확인필요 | 명단(7번)에 더 적혀 있는데 제출 안 됨 → `Y` |
| 비고 | 확인필요 사유 + 원본 명단 텍스트 |

> 합계 검산: `Σ(1인등록비+본인버스+본인설악산) + Σ그룹공동비용 = 전체 총액` (중복 없음)

## 실데이터 검증 결과 (회사 사본, 185명 기준)

- 그룹 92개 (가족/그룹 31 + 1인 61)
- **전체 합계 66,738,000원** (인당합+공동합 검산 일치 ✔)
- 대표자 추정 91/92 정상
- **확인필요 36건** 자동 탐지 (명단 > 실제 제출 — "황성수 미제출" 류 케이스)

## 컬럼 매핑 (헤더 텍스트로 자동 탐색 — 순서 바뀌어도 동작)

이메일 / `등록자 이름`(첫번째) / `그룹의 대표자` / `소속부서` / `객실` /
`가족/그룹을 신청` / `명단` / `버스 신청 혹은 자차` / `설악산뷰` / `입금자명`(첫번째)

> ⚠️ `대표자`만으로 찾으면 col3(등록자 이름 설명문에 '대표자' 포함)에 잘못 걸리므로
> `그룹의 대표자`로 매칭한다. (검증 중 발견·수정한 포인트)

## 한계 (의도된 동작)

- 그룹 묶기는 **이메일** 기준. 부부가 다른 이메일로 따로 내면 별도 그룹 → 명단/입금자명으로 `확인필요` 표시.
- 명단은 자유 텍스트라 인원수 추정이 완벽하지 않음 → **숫자는 참고용, 원본 명단을 비고에 그대로** 실어 사람이 검토.
- 이 한계들은 통화에서 합의된 "애매하면 사람이 확인" 방침과 동일.
