# 2026 전교인 리트릿 등록 원본 데이터 해석 가이드

> **이 문서가 데이터 해석의 표준(spec)입니다.** 웹앱/Apps Script는 이 기준을 따릅니다.
> 핵심: **구글폼 원본 응답(`설문지 응답 시트1`)은 절대 수정하지 않고, 해석 레이어에서 정규화/보정/예외처리한다.**

## 1. 기본 원칙

### 1) 원본 데이터는 수정하지 않는다
`설문지 응답 시트1`은 원본 보존용. 웹앱은 원본을 직접 수정하지 않고 별도 해석/정규화 레이어에서 아래 기준을 적용.

### 2) 열 번호가 아니라 반드시 헤더명 기준으로 읽는다
원본 시트는 중간에 열이 추가됐고 앞으로도 바뀔 수 있음 → `A열` 같은 고정 인덱스 금지, **헤더 텍스트 기준**.

| 의미 | 원본 헤더 |
|---|---|
| 구버전/중복 제외 여부 | `구버전 여부` |
| 등록자 이름 | `1. 등록자 이름 (그룹의 경우, 대표자 이름은 아래 2번에 따로 적어주세요)` |
| 성별 | `1-1. 등록자 성별` |
| 연락처 | `1-2. 연락처` |
| 대표자 이름 | `2. 그룹의 대표자 이름 ...` |
| 캠퍼스 | `3. 주로 예배 드리시는 캠퍼스를 표시해주세요` |
| 부서/개인 등록비 | `4. 등록자의 소속부서를 선택해주세요 [개인 비용]` |
| 객실 선택 | `5. 원하시는 객실의 종류...` |
| 그룹/부분/개인 선택 | `6. 가족/그룹을 신청하시겠습니까? [그룹 비용]` |
| 명단 | `7. 가족/그룹을 신청하는 경우... 명단...` |
| 교통 | `8. 등록자의 버스 신청 혹은 자차 이용 여부...` |
| 설악산뷰 | `9. 설악산뷰 선택 [개인 비용]` |
| 입금자명 | `10. 입금자명을 적어주세요` |
| 문의사항 | `11. 궁금하신 문의사항...` |

## 2. 구버전/중복 처리
`구버전 여부`가 `구`/`중복`/유사 의미면 **집계 제외**(참고만). 같은 사람 여러 번 신청 시 ① `구버전 여부` 빈 최신 행 우선 ② 새 그룹 합류 시 기존 개인 신청은 제외/이동 ③ 원본 행은 삭제·수정 금지, 해석 결과에만 반영.

## 3. 신청 유형 분류 (그룹/개인/부분)
- **그룹**: `6번`에서 "N인이 투숙" 선택 / 명단에 방 전체 멤버 / 가족·셀·청년부 한 방 확정 요청
- **개인**: "교회에서 정해주시는 대로" / 명단 없음 / 혼자 / 객실 희망만 있고 동행 미정 (객실 희망은 보존)
- **부분**: "부분적으로 그룹을 원합니다" / "저와 OO 2명 함께 배정" / 일부만 묶어달라는 요청
  - 부분은 최종 방배정 전까지 `개인/부분 교회 배정` 현황에 포함, 그룹 확정 객실 통계에서는 제외.

## 4. 비용 계산 (만원 단위)
그룹 비용은 "그룹당 1회". 그룹 총액 = 각 개인 기본등록비 합 + 버스비 합 + 설악산뷰 + 객실선택비 + 그룹비.

### 4-1. 기본 등록비
장년 27.8 / 청년 27.8 / 중고등 26.8 / 소년 25.8 / 초등 24.8 / 유년 22.8 / 유치 20.8 / 영유아 19.8 / 영아돌전 17.8
(원본 문항에 금액+설명 포함 → 키워드 파싱. 예 `장년부 278,000원`→장년/27.8)

### 4-2. 버스비
자차 0 / 버스 3.8(1인). 개인=1×3.8, 그룹=실제 버스 인원×3.8, 부분=부분 내 버스 인원×3.8.
**주의: 그룹이라고 전원 버스 아님.** 개별 신청서의 각자 교통 선택 우선.
예) 김은학 그룹 김지민B 버스·김경은 자차 / 김연지 그룹 김윤하 자차 / 이혜란만 부산버스·나머지7 분당자차 / 차윤주 부산버스.

### 4-3. 설악산뷰
1(1인). 개인=1×1, 그룹=방 전체×1, 부분=부분 인원×1. 신청 희망값으로 보존.

### 4-4. 객실선택비
| 객실 | 개인 | 그룹 |
|---|--:|--:|
| 소노벨 패밀리 | 0 | 0 |
| 소노벨 스위트 | 1 | 6 |
| 소노캄 스위트 | 4 | 24 |
같은 객실이라도 개인/그룹에 따라 다름. 부분은 최종 전까지 참고용.

### 4-5. 그룹비 (그룹당 1회)
7~8인 0 / 6인 5 / 5인 10 / 4인 20 / 3인 30 / 2인 40 / 1인 50. 대표자 이름으로 합산 납부.

## 5. 침구추가 / 기본인원 초과
기본 인원: 소노벨 패밀리 4 / 소노벨 스위트 5 / 소노캄 스위트 5.
`침구추가 = max(0, 투숙인원 - 기본인원)`. **그룹 확정 객실에만** 적용(개인/부분 공란).

## 6. 이름 정규화
| 원본/상황 | 정규화 |
|---|---|
| `첼로09` | 박윤정 |
| `JONGJIN JUNG` | 정종진 |
| `이한나` 청년부 | 이한나A |
| `이한나 B`(임창은 자녀) | 이한나B |
| 김은학 그룹 김지민 | 김지민B |
| 임진수/김리아 가족 김지민(추후) | 김지민A |
| 김상아 가족 이사랑 | 이사랑B |
| 박은미 부분 이사랑 | 이사랑A |
| 홍규화 그룹 홍민정 | 홍민정(청년부) |
원칙: 앞뒤 공백 제거 / 괄호 설명 제거(필요시 비고 보존) / 동명이인 A·B suffix 유지 / 원본 보존 가능, 표시용은 정규화.

## 7. 신청서 제출 여부 / 추정 인원
명단엔 있으나 개인 신청서 없는 사람은 **제거하지 말고** 표시:
| 상태 | 처리 |
|---|---|
| 제출 확인 | 정상, 검정 |
| 명단 있으나 신청서 없음 | 인원 포함, 상태 `미제출/추정` |
| 부서 불명 | 장년/청년 27.8 임시 or `부서 확인 필요` |
| 교통 불명 | 대표자 기준/자차 임시, `교통 확인 필요` |
표현: 미제출/추정 파랑, 비고에 사유.

## 8. 통계 기준
### 8-1. 방배정 분류 통계 (그룹+개인+부분)
최신 기준: 그룹 41방/199명, 개인 22/22, 부분 5/11, **전체 68/232**.
### 8-2. 그룹 신청 객실 통계 — **그룹만** (개인/부분 제외, 방 미확정).
### 8-3. `개인/부분 교회 배정 신청 현황` — 개인+부분을 성별·희망객실로. 합계=개인+부분 일치. 남녀혼합 부분은 실제 분리. 0이면 메모 안 남김. 최신: 남15/여18/계33.
### 8-4. `개인/부분 방배정에 필요한 객실 수 예상` — `ceil(성별합계/정원)`, 남녀 분리 기본.
### 8-5. `개인/부분 신청 시 희망한 객실 (참고용)` — 희망 분포 참고, 최종과 다를 수 있음.

## 9. 캠퍼스/교통
분당/부산/부산·분당 혼합. 혼합 그룹은 대표 캠퍼스만으로 처리 금지, 사람별 보정:
이혜란(이혜란만 부산/7명 분당) / 김연지(차윤주만 부산) / 조형만(혼합·자차4) / 정석현(부산4 자차) / 김남현·문상철·길태형(부산청년 부분·버스3).

## 11. 최신 추가 반영
정석현 가족(정석현·홍은희·정해이·정시하 부산4 그룹) / 박수림(개인·패밀리·분당·버스) / 김창준(개인·패밀리·분당·자차) / 이은희(개인·패밀리·분당·자차) / 김지민B·김경은(김은학 그룹 제출) / 김윤하(김연지 그룹 제출·자차).
**주의: 김윤하는 김연지 그룹 명단에 이미 있어 개인 중복 추가 금지.**

## 12. 검증/경고 플래그
그룹인데 명단 없음 / 명단≠선택인원 / 부분 남녀혼합 / 그룹 내 교통 일부 다름 / 명단 있으나 신청서 없음(미제출) / 부서 불명 / 대표자명≠입금자명 / 동명이인 / 구버전 / 개인·부분인데 그룹비 산출(참고값 표시).

## 13. 해석 순서
1.원본 로드 2.헤더 매핑 3.구버전/중복 제외 4.이름 trim/정규화 5.유형판정(그룹/개인/부분) 6.대표자추정·그룹ID 7.명단파싱 8.개별 오버라이드 9.부서/성별/캠퍼스/교통/객실 정규화 10.비용계산 11.미제출/추정 표시 12.통계 13.경고생성.

## 14. 핵심
> 원본 = "사용자가 적은 신청 내용", 웹앱 결과 = "교회 운영 기준으로 해석된 정규화 데이터".
단순 행 합산 금지. **구버전 제외 · 이름 정규화 · 그룹 병합 · 부분 분리 · 미제출 보존 · 개별 예외** 필수.
