성도용 (1 ~ 30)

1입금 문구 친절하게완료
요청입금 안내 하단 문구를 더 부드럽게
적용→ "원활한 등록 관리를 위하여 위 항목별로 구분하여 따로 입금 부탁드립니다"로 변경
위치 등록 → 입금 안내 / 복사 문구 / 접수 이메일
2복사문자에 입금자명·금액 제목 달기완료
적용표 첫 줄을 ▸ 입금자명 금액 (설명) 헤더로, 각 줄은 ▸ 등록 김바울 278,000원 (장년부) 형식
추가목사님 요청으로 헤더의 [항목] 글자는 제거 — 성도가 입금자명에 이름만 쓰지 않도록
위치 입금 안내 · 복사 · 이메일 공통
3설명용 예시 이미지완료
적용입금자명 예시 이미지 + "🧮 비용 예시 보기"(상황별 탭형 안내) + 객실 평면도까지 추가
위치 등록 화면 상단 / 방 선택
4구성원별 캠퍼스 구분완료
적용그룹 신청 시 구성원마다 캠퍼스 선택 버튼 노출 (미선택 시 대표자와 동일)
위치 그룹 등록 → 구성원
5대표자 → 1번 구성원 자동입력완료
현재1번에 "(대표자)"만 있고 실제로는 비어 계산에서 누락
적용→ 대표자 이름을 입력하면 1번 구성원에 자동으로 채워짐
위치 그룹 등록 → 대표자/구성원
6버스·설악 비용 개인/그룹 입금 선택완료
적용"각자 등록비 입금" 선택 시, 버스비·설악산을 개인별 합산 vs 대표자 일괄 라디오로 선택 → 입금 안내가 그에 맞게 분기
위치 그룹 등록 → 입금 방식
7부분 그룹 체크박스 신설완료
적용"나머지 멤버는 교회에서 배정해주시면 좋겠습니다" 체크박스 + 체크 시 인원 버튼 비용 표시를 "추후 결정됨"으로
위치 그룹 등록 → 몇 분이 한 방을
8부분 그룹 시 객실비 개인비용 처리완료
적용부분그룹 체크 시 객실 선택을 1인 기준(스위트 1만·소노캄 4만)으로 안내, 최종 입금은 그룹 기준으로 처리
위치 그룹 등록 → 방 선택 / 입금 안내
9제출 버튼 문구완료
변경"제출하기" → "저장하고 제출하기"
10입금 안내 복사 버튼 한번 더 노출완료
적용"입금 안내 문구 복사하기" 버튼을 "저장하고 제출하기" 위에 한 번 더 노출
11설악산뷰 그룹 공통화완료
변경구성원별 설악산뷰 → 그룹 전체 공통 (전원 적용 · 1인 1만원)
위치 조회·수정 / 구성원
12·13·14수정화면 버튼색 일관성완료
적용객실/투숙 저장·이 내용으로 수정 저장 = 검정(제출하기와 통일). 구성원 추가 = 점선 테두리 + 연빨강 채우기
위치 조회·수정 화면
15수정화면 부분그룹 정보(확정/미정)완료
적용확정/미정 토글 + 미정이면 인원 옆 "(추가비용 추후결정됨)" + "나머지 인원은 교회에서 배정" 안내
위치 조회·수정 → 투숙 인원
16데이터 버전관리 — 수정 시 새 행완료
적용덮어쓰기 대신 새 행 생성(새 타임스탬프) + 기존 행 "구버전 여부"에 "구" 표시 (이력 보존)
위치 응답 시트 / 백엔드
17구성원 삭제 마크완료
적용행을 지우지 않고 "앱에서 삭제신청"으로 표시(값 보존·집계 제외)
18신청 결과 이메일 발송완료
적용접수 시 항목별 입금 안내가 담긴 메일 자동 발송 (발신주소는 send-as 별칭 등록 후 설정)
19보안 강화 — 이름+이메일 본인확인완료
변경전화번호 → 이메일 기준 본인확인 (조회·수정·삭제)
20구성원 수정 시 인원 자동반영완료
적용확정 상태에서 구성원 추가/삭제하면 투숙 인원이 자동으로 맞춰짐
위치 조회·수정 → 객실·투숙인원
21미정 상태 인원 선택 활성화완료
적용미정(부분)이어도 희망 인원을 선택해 기록 가능(다음 조회 때 유지)
22구성원 추가 시 캠퍼스 선택완료
적용수정화면에서 구성원 추가 시 캠퍼스 선택 버튼 노출
23대표자 이름·구분 혼선완료
현재박테스트1/4 조회 시 대표자가 "박테스트"로 잘리고 엉뚱한 사람이 대표
적용→ 대표자 추정 규칙에 접미사·숫자 보존(박테스트1 유지) + 이메일·전화 그룹 키에 대표자 결합해 오병합 방지
24수정 결과 메일링완료
적용수정·구성원 추가·삭제·그룹설정 변경 시 각각 안내 메일 발송
25미정 탭다운 "인원 상관없음"완료
적용수정화면 미정 모드에 "인원은 상관없습니다" 버튼 추가
26수정 메일 디테일완료
현재요약만 있고 무엇이 바뀐지 없음
적용→ 변경된 항목(무엇 → 무엇) + 변경 후 최종 입금 내역(항목별·총액) 둘 다 포함
27객실/투숙/설악 저장도 버전관리완료
적용그룹설정 저장도 새 행 생성 + 기존 행 "구" 표시
28저장 버튼 텍스트완료
변경"객실/투숙/설악산뷰 저장" → "객실/인원/설악산뷰 저장"
29대표자 삭제 시 처리완료
적용대표자를 삭제하면 남은 구성원 중 새 대표자를 고르는 팝업이 떠서, 선택해야 삭제 진행
30메일 비전·영적 문구문구 대기
적용관리자 ✉️ 메일문구 탭에 "비전·영적 기대 문구" 칸을 만들어 둠 → 적으면 모든 안내 메일 끝에 자동 삽입
남음목사님/사장님이 문구만 주시면 바로 반영

관리용 (31 ~ 34)

31+방 객실 타입 선택완료
적용방배정 "+ 방 추가"를 드롭다운으로 — 패밀리/소노벨 스위트/소노캄 스위트 선택해 생성(빈 방도 정원·타입 유지)
위치 관리자 → 방배정
32배정된 방의 객실 종류 변경완료
적용방배정 보드의 각 방 헤더에 타입 드롭다운 → 바꾸면 그 방 인원 전원의 배정방이 새 타입으로(저장 시 반영), 정원·경고도 갱신
위치 관리자 → 방배정 → 각 방
33미배정 인원 분류완료
적용미배정 풀을 남/여 분리 + 정렬(ㄱㄴㄷ·부서순) + 객실 필터(패밀리·스위트·소노캄)
덤네임태그 남=파랑·여=분홍 색구분 / 한 방에 남녀 혼방 드롭 차단
위치 관리자 → 방배정 → 미배정
34수정 시 가격 미리보기완료
현재수정 전 금액으로 고정
적용→ 객실·인원·설악·구성원 바꾸면 저장 전에도 "예상 그룹 총액"이 항목별·총액으로 실시간 갱신
위치 조회·수정 → 저장 버튼 위

리스트엔 없지만 이번에 함께 추가/개선

📋 정리안 = 운영자 전수 분석본
현재 신청 전체를 그룹/부분/개인/중복/의사결정으로 분석해 표시. 정렬(접수·ㄱㄴㄷ·인원순) + 🤖 AI 정리안 갱신 버튼(API 키 넣으면 셀프 분석).
⚠️ 체크필요 탭 + 이슈관리 시트 (sv21)
충돌(미제출·객실 불일치·부분그룹)을 구글시트 '이슈관리' 탭에 스캔·기록. 상태(미해결/확인중/해결)·메모 관리, 해결되면 "현재 미감지" 자동 표시. 시트·관리자 양쪽에서 편집.
해당 성도가 조회·수정 들어오면 확인 배너로 상황 정리 + "대표 한 분이 통일해 수정" 안내.
✉️ 메일문구 탭
접수·수정·추가·취소·그룹변경 메일의 제목·본문을 관리자가 직접 편집 + 전체 미리보기.
📐 객실 평면도
방 선택 시 객실별 고화질 평면도(사진+평면도) 표시 + 탭하면 두 손가락 핀치 확대.
🛡️ 데이터 안전
enrich 재실행 시 관리자 칸(배정방·입금확인·메모)을 덮지 않고 보존 — 수기 방배정이 날아가던 문제 수정(v17).
'테스트' 표시 행은 조회엔 보이되 모든 집계에서 제외.
