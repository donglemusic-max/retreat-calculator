디자인팀 검토

2026 리트릿 등록 — UI/UX 전면 검토

기능이 늘며 화면이 복잡해진 상태를 진단하고, 가독성·직관성을 높이는 디자인 시스템과 화면별 개선안을 제안합니다.

한 줄 결론

지금 화면은 "규칙이 없어서" 복잡합니다. 폰트 16종·회색 4종·파랑 3종이 섞이고 모든 섹션이 항상 펼쳐져 있어, 무엇이 중요한지가 안 보입니다. 해법은 새 화면이 아니라 ① 타입 스케일 5단계 ② 컬러 역할 정리 ③ 점진적 노출(접기) ④ 8px 간격 그리드 4가지 규칙을 일괄 적용하는 것입니다.

2. 제안 디자인 시스템 (4규칙)

① 타입 스케일 — 16종 → 6단계

역할별로 크기를 고정합니다. "그냥 조금 작게"를 금지.

Display 28/800총 등록 금액
Title 19/700카드 제목
Body 15/400기본 본문·입력값
Label 14/700버튼·필드 라벨

※ 11px 이하는 폐기(모바일 가독 한계). 최소 본문 13px.

텍스트 회색은 3단계(제목/본문/보조)로만. 파랑은 1개 + 진하기 2단계. 빨강은 오류·삭제에만(현재 "구성원 추가"가 빨강인 건 의미 충돌 → 점선+중립색 권장).

③ 점진적 노출 — 기본은 접어두기

  • 자동값이 있는 건 접기: 투숙 인원(인원수 자동), 입금 방식 상세, 예시 이미지는 기본 접힘 + "필요하면 펼치기".
  • 한 화면 = 한 결정: 스크롤 한 구간에 핵심 1개만. 보조 안내는 ⓘ 도움말로.
  • 금액은 항상 하단 고정 요약 1곳으로 모음(여러 카드에 흩어진 금액 표기 정리).

④ 간격·그리드 — 8px 리듬 + 모서리 2종

  • 간격은 4·8·12·16·24·32만 사용(들쭉날쭉 padding 제거).
  • 모서리: 카드 16px / 버튼·입력 12px 2종으로 통일(현재 lg·xl·2xl 혼용).
  • 카드 외곽선 1px + 아주 옅은 그림자 1종 → 카드 간 경계는 명확하되 무겁지 않게.

3. 화면별 개선안

개인 / 그룹 등록

지금

  • 방 선택·투숙·설악·입금방식 카드가 모두 펼쳐져 길다
  • 카드마다 금액 표기가 달라 합계 추적이 어려움
  • 버튼 색(파랑/검정/빨강)이 규칙 없이 섞임

개선

  • 투숙·입금 상세는 기본 접기(자동값 노출 + "변경")
  • 하단 고정 금액 요약 바 1곳으로 합계·1인 평균 집약
  • 주요 액션=Brand, 보조=중립 외곽선, 위험=Bad로 일관

조회 · 수정

지금

  • 구성원 카드마다 동일 필드가 반복돼 길고 빽빽
  • "내가 입금할 금액"·"그룹 총액"·"공동비용" 용어가 함께 떠 혼란

개선

  • 구성원은 요약 행(이름·부서·금액)으로 접고, 탭하면 펼쳐 수정
  • 금액 용어 2개로 축소: "내 입금액" + "그룹 합계(대표자)"

관리자

지금

  • 탭·통계·보드·리스트가 한 번에 많음(운영자용이라 밀도 OK지만 라벨 모호)

개선

  • 밀도는 유지하되 섹션 타이틀 + 색 코딩(미입금=warn, 확정=good)으로 스캔성↑

4. 적용 우선순위

순위작업효과 / 리스크
P0타입 스케일 6단계 + 회색 3·파랑 1 토큰화(공용 클래스/상수)전 화면 즉시 정돈 / 낮음(시각만)
P0모서리·간격 토큰 통일(12/16, 8px 리듬)정돈감↑ / 낮음
P1점진적 노출: 투숙·입금상세·예시 기본 접기길이 체감 30~40%↓ / 중간
P1하단 고정 금액 요약 바 + 금액 용어 정리직관성↑ / 중간
P2조회 구성원 카드 → 접이식 요약 행조회화면 정돈 / 중간

권장 진행

P0(토큰화)부터 적용하면 코드 변경은 크지만 리스크가 낮고(시각 일관성만 바뀜) 전 화면이 한 번에 정돈됩니다. P1·P2는 동작이 바뀌므로 배포 후 화면 보며 단계 적용을 권합니다.

이 페이지 자체가 제안한 규칙(타입 6단계·회색 3·파랑 1·8px 리듬)으로 그려졌습니다 — 같은 결을 앱에 이식하는 작업입니다.
